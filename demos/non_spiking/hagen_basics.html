The non-spiking mode¶

This example presents the non-spiking mode of the BrainScaleS-2 ASIC and some of its characteristics. The operation of this so-called hagen mode is explained in more detail at the end of the single neuron tutorial.

%matplotlib notebook
import torch
import hxtorch

import matplotlib as mpl
import matplotlib.pyplot as plt

from IPython.display import clear_output
from ipywidgets import interact, IntSlider
from functools import partial
IntSlider = partial(IntSlider, continuous_update=False)

%%html
<style>.output_wrapper button.btn-default, .output_wrapper .ui-dialog-titlebar {display:none}</style>

def get_sliders(weight=True):
    sliders = dict(
        num_inputs=IntSlider(128, 1, 256),
        input_value=IntSlider(12, 0, 31),)
    if weight:
        sliders.update(weight_value=IntSlider(21, -63, 63))
    return sliders

The hxtorch API¶

The hagen mode provides an analog multiply accumulate operation (MAC) which is performed on the ASIC.
hxtorch, introduced in the [hxtorch paper], represents the high-level API. In analogy to some functions of the machine-learning framework PyTorch, operations with similar API are provided, e.g. matmul for multiplication of two matrices:
print(hxtorch.matmul.__doc__)

Before the hardware can be used, a connection must be created and a default calibration must be loaded using hxtorch.init:

# initializes the hardware connection
# and applies a default calibration
hxtorch.init()

Noise and fixed-pattern deviations¶

Despite calibration and even with the same inputs and weights, the outputs of the different neurons are not identical. On the one hand, each output has a statistical noise due to the analog nature of the neuron, on the other hand, fixed-pattern deviations show up between the individual neurons.
Especially in the case of small inputs, a spatial correlation may also become apparent, resulting from different distances to the synapse drivers.
fig, axes = plt.subplots(1, 2, sharey=True)
axes[0].set_ylim(-130, 130)
neurons = torch.arange(0, 256)
slices = [slice(128, 256), slice(0, 128)]
lines = list()
for ax, s in zip(axes, slices):
    lines.append(ax.plot(neurons[s], torch.zeros(128))[0])
    ax.set_xlabel("neuron #")
axes[0].set_ylabel("output")
axes[0].invert_xaxis()

@interact(**get_sliders())
def update(num_inputs, input_value, weight_value):
    """ Updates the plot with the outputs from the hardware """
    out = hxtorch.matmul(
        torch.full((1,), input_value, dtype=torch.float),
        torch.full((1, 256), weight_value, dtype=torch.float),
        num_sends=num_inputs)
    for line, s in zip(lines, slices):
        line.set_ydata(out[s])

Linearity of the MAC operation¶

The next plot shows the linear relationship between input, weight and output. For this purpose, a constant input is multiplied by a linearly increasing weight vector.

plt.figure()
plt.ylim(-130, 130)
weight = torch.arange(-63, 64.).repeat_interleave(2)
line, = plt.plot(weight, torch.zeros_like(weight))
plt.xlabel("weight"); plt.ylabel("output");

@interact(**get_sliders(False))
def update(num_inputs, input_value):
    """ Updates the plot with the outputs from the hardware """
    out = hxtorch.matmul(
        torch.tensor(input_value, dtype=torch.float).unsqueeze(-1),
        weight.unsqueeze(0),
        num_sends=num_inputs)
    line.set_ydata(out)

At output values of about -80 to 80 a good linear correlation can be observed. For smaller or larger values, the used ADC saturates; this happens earlier for some neurons and later for others.

# this releases the hardware recources
hxtorch.release()
The integration with PyTorch allows the MAC to be used very easily for conventional machine learning. For this, the forward pass is computed on the ASIC, the backward pass on the host computer.
The Yin-Yang example shows such a usage.
